# Test Plan – Restaurant Management System

## 1. Objective
The purpose of this testing is to validate the functionality, reliability, and performance of the restaurant management system, focusing on reservations, table allocation, and user workflows.

## 2. Scope

### In Scope:
- Reservation creation
- Table assignment
- Booking validation (date, time, guests)
- Double booking prevention
- User roles (admin/staff)

### Out of Scope:
- Payment processing
- Third-party integrations

---

## 3. Test Strategy

### Testing Types:
- Functional Testing
- UI Testing
- API Testing
- Regression Testing

### Levels:
- Manual Testing
- Automated Testing (Cypress)

---

## 4. Test Environment
- Browser: Chrome
- OS: Windows / macOS
- Tools:
  - Cypress (UI Automation)
  - Postman (API Testing)
  - GitHub (Version Control)

---

## 5. Entry & Exit Criteria

### Entry Criteria:
- Application is accessible
- Requirements are defined

### Exit Criteria:
- All critical test cases executed
- No high severity bugs open

---

## 6. Risks
- Unstable test environment
- Incomplete requirements
- Limited test data

---

## 7. Deliverables
- Test Cases
- Bug Reports
- Automation Scripts
- API Test Collections
